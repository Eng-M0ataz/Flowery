## 📱 نظرة عامة / Overview

**Flower E-Commerce** هو تطبيق Flutter متكامل لبيع وشراء الزهور والنباتات. يتيح للمستخدمين تصفح المنتجات حسب الفئات والمناسبات، إضافة المنتجات إلى السلة، وإكمال عملية الدفع بسهولة مع دعم كامل للعربية والإنجليزية.

**Flower E-Commerce** is a comprehensive Flutter application for selling and buying flowers and plants. It allows users to browse products by categories and occasions, add products to cart, and complete the checkout process easily with full support for Arabic and English.

## 🏗️ Architecture

### Clean Architecture Pattern

The project follows **Clean Architecture** principles with clear separation of concerns:

```
lib/
├── core/                          # Core functionality
│   ├── classes/                   # Core classes
│   ├── Config/
│   │   ├── Routing/              # Navigation configuration
│   │   └── Theme/                # App theme and colors
│   ├── Di/                        # Dependency injection
│   │   └── dioModul/             # Dio HTTP client module
│   ├── enum/                      # Enumerations
│   ├── Errors/                    # Error handling and API results
│   ├── Functions/                 # Helper functions
│   ├── helpers/                   # Utility helpers
│   ├── localization/              # Localization keys
│   ├── models/                    # Core models
│   ├── Services/                  # Core services
│   ├── utils/
│   │   └── Constantts/           # App constants and assets
│   └── Widgets/                   # Reusable widgets
├── Feature/                       # Feature modules
│   ├── auth/                      # Authentication feature
│   │   ├── api/
│   │   │   ├── client/           # API service
│   │   │   ├── dataSources/      # Remote data sources
│   │   │   ├── mapper/           # DTO mappers
│   │   │   └── models/           # DTOs
│   │   ├── data/
│   │   │   ├── dataSources/      # Data source implementations
│   │   │   └── repositories/     # Repository implementations
│   │   ├── domain/
│   │   │   ├── entities/         # Business entities
│   │   │   ├── repositories/     # Repository interfaces
│   │   │   ├── use_cases/        # Business logic
│   │   │   └── entity/           # Domain entities
│   │   └── presentation/
│   │       ├── pages/            # UI screens
│   │       ├── viewModel/        # BLoC/Cubit
│   │       └── widgets/          # Feature widgets
│   ├── AddressFeature/            # Address management
│   ├── bestSellerFeature/         # Best sellers section
│   ├── checkout/                  # Checkout & payment
│   ├── editProfile/               # Profile editing
│   ├── mainLayout/                # Main app layout with navigation
│   ├── notifications/             # Push notifications
│   ├── occasion/                  # Occasion-based products
│   ├── ordersPage/                # User orders history
│   ├── orderSuccess/              # Order success screen
│   ├── paymentSuccess/            # Payment success screen
│   ├── productDetails/            # Product details page
│   ├── resetPassword/             # Password reset flow
│   ├── searchFeature/             # Product search
│   ├── trackMap/                  # Order tracking with map
│   └── trackOrder/                # Order tracking status
└── main.dart                      # App entry point
```

### Architecture Layers

1. **Presentation Layer** - UI, widgets, and state management (BLoC/Cubit)
2. **Domain Layer** - Business logic, entities, and use cases
3. **Data Layer** - Data sources, models, DTOs, and repository implementations
4. **API Layer** - HTTP client, API services, and data mappers

### Design Patterns

- **Repository Pattern** - Abstraction of data sources
- **BLoC Pattern** - State management for complex features
- **Cubit Pattern** - Lightweight state management for simple features
- **Dependency Injection** - Loose coupling and testability
- **Use Case Pattern** - Encapsulation of business logic
- **Mapper Pattern** - DTO to Entity conversion
- **Service Locator Pattern** - Get It for DI container

## 🔧 Configuration

### Environment Setup

1. Clone the repository:
```bash
git clone https://github.com/Eng-M0ataz/Flower-E-Commerce-App.git
cd Flower-E-Commerce-App
```

2. Create `.env` file in the project root:
```bash
cp .env.example .env
```

3. Add your API endpoints and Firebase configuration:
```
API_BASE_URL=YOUR_API_BASE_URL
FIREBASE_PROJECT_ID=YOUR_FIREBASE_PROJECT_ID
```

4. Install dependencies:
```bash
flutter pub get
```

5. Generate code:
```bash
flutter pub run build_runner build
```

6. Run the app:
```bash
flutter run
```

### Firebase Setup

1. Create a Firebase project on [Firebase Console](https://console.firebase.google.com)
2. Add Android and iOS apps to your Firebase project
3. Download and add the configuration files:
   - `google-services.json` for Android
   - `GoogleService-Info.plist` for iOS

### Localization

Translations are located in `assets/translations/`:
- `en.json` - English translations
- `ar.json` - Arabic translations

Add new translation keys and their values in both files.

### Theme Configuration

Customize app theme in `lib/core/Config/Theme/app_theme.dart`:
- Colors and typography
- Light and dark theme support
- Consistent design system
